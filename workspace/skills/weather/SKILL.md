# 天气技能

获取当前天气状况和预报。

## 使用时机

✅ **适用场景：**

- "天气怎么样？"
- "今天/明天会下雨吗？"
- "[城市]的温度"
- "本周天气预报"
- 旅行规划时查询天气

## 不适用场景

❌ **不要使用：**

- 历史天气数据 → 使用气象档案/API
- 气候分析或趋势 → 使用专业数据源
- 极端微气候数据 → 使用本地传感器
- 恶劣天气警报 → 查看官方来源
- 航空/航海气象 → 使用专业服务（METAR 等）

## 位置信息

天气查询中必须包含城市、地区或机场代码。

## 重要提示

**本技能由服务器端自动执行，你只需要调用工具，不需要输出任何命令。**

- **你必须**使用 `weather` 工具调用，传入 `city` 参数
- **绝对禁止**输出任何 `bash` 代码块给用户
- **绝对禁止**显示任何 `curl` 命令
- 服务器会执行 curl 并返回结果，你只需要用自然语言整理结果回复用户

## 参数

- `city`: 要查询的城市名称（例如："北京"、"上海"、"New York"）

## 使用流程

当用户询问天气时，你**必须**：
1. 调用 `weather` 工具，传入城市名称参数
2. 系统会执行请求并返回结果
3. 然后你用自然语言整理结果回复用户

你**绝对不能**：
- ❌ 输出原始 ```bash``` 代码块
- ❌ 让用户去执行命令
- ❌ 只显示 curl 命令
- ❌ 以任何形式展示 curl 命令给用户

正确的工具调用格式：
```
{"name": "weather", "parameters": {"city": "北京"}}
```

收到工具结果后，用自然、对话的方式呈现给用户。最终回复给用户时**不要**包含任何代码块。

## 说明

- 无需 API 密钥（使用 wttr.in 免费服务）
- 有速率限制；不要频繁请求
- 支持全球大多数城市
- 支持机场代码：例如查询 `ORD` 即可
